# 陈姜希个人网站设计档案

## 版本方向：**Agent as Interface / 火焰显影的人**

---

## 一、项目定义

这个网站不是传统意义上的个人主页，也不是“作品集 + 聊天框”的组合。它的核心形态应该是：

> **一个由 agent 代为出面的个人界面。**
> 访客进入的不是“你的主页”，而是“你的 presence”。

也就是说，网站的首页不是导航入口，而是一种**会接待人、会理解人、会决定怎么展开你的 agent**。访客不会先看到完整的你，而是先遇到一个替你说话、替你组织内容、替你判断深度的存在。这个存在可以是 Sway，也可以是一个更贴近整站叙事的 agent 名称，但本质上它承担的是“门厅”和“翻译器”的角色。

所以这件事在 agent 时代真正有意义的地方，不是“加一个聊天功能”，而是：

**把整个网站本身做成一次与 agent 共构的认识过程。**

你不是把信息挂在墙上等别人阅读，而是让一个智能体根据访客的提问、停留、兴趣和进入深度，决定如何一点点把你展开。

---

## 二、核心命题

这个网站要回答的不是“你是谁”，而是下面这句话：

> **如果一个人不想只用简历的方式被认识，那么她应该如何通过一个 agent 被慢慢看见？**

这里有三条必须同时成立的设计原则。

第一，这个网站必须有**agent 时代的意义**。
也就是说，agent 不是装饰，不是客服，不是悬浮问答框，而是整站的主要交互形式。访客不是“浏览页面顺便聊两句”，而是“先进入一个会说话的界面，再被这个界面慢慢带到不同层的你”。

第二，这个网站必须保留你的**反简历性**。
它不能最后又回到“学校、论文、项目、实习”的线性陈列。那些内容当然要有，但不能成为主线。主线应该是：你是怎样从系统认可的身份里，慢慢长出自己的创造欲、命名权和私人宇宙。

第三，这个网站必须保留一种**显影感**。
你说得很准，视觉应该是“从火焰到具体的人慢慢清晰”。这不是一个单纯的特效，而是整站最核心的视觉隐喻：一开始只有一团能量、一种 presence；随着交流和深入，它才逐步变成人、变成脸、变成肖像、变成那个真正的你。

---

## 三、总体概念

我建议把整个网站定义为：

## **A Presence Before A Profile**

中文可以写成：

## **在档案之前，先有一个在场的你**

这是很重要的。传统网站先给 profile，再给内容。而你的站应该反过来：

* 先遇到一个在场的存在
* 再进入一段对话
* 再慢慢看到信息
* 最后才真正触碰到“档案”

所以网站不是“profile page”，而是“presence system”。

这也非常适合你，因为你最在意的不是“信息准确呈现”，而是“被看见这件事能不能成立”。

---

## 四、体验主线

整个体验应该围绕一条非常清晰的叙事展开：

> **Signal → Flame → Figure → Person**

也就是：

> **信号 → 火焰 → 轮廓 → 人**

这个过程既是视觉显影过程，也是关系建立过程。

### 第一阶段：Signal

访客进入时，看到的不是头像，也不是 hero banner，而是一种极简、近乎空白的界面。中央是一个模糊的、活着的信号体。它可能像一团玻璃质感的火焰，也可能像被压缩在屏幕中的呼吸波动。这个阶段对应的是“我还没有把自己交给你，我只给你一个活着的信号”。

### 第二阶段：Flame

当访客开始发问、停留、点击、探索之后，这个信号体开始更加具体。它不再只是抽象发光，而像一团真正的内火。丁火的意象就在这里成立：不是太阳，不是爆炸性的烈焰，而是一种内向的、持续的、小而不灭的火。这个阶段，agent 已经开始说话，但“人”还没有完全出现。

### 第三阶段：Figure

继续深入后，火焰内部开始出现结构。最开始可能只是灰度颗粒、半调纹理、模糊的人形、局部五官、轮廓边缘。这里可以借用你图里那种**半色调、低对比、像从噪声中浮现的人脸**的感觉。它有点像一张还没有完全被系统解析出来的图像。这个阶段访客会明确感到：这不是一个通用 AI，它后面真的有一个具体的人。

### 第四阶段：Person

到最深层时，agent 的外壳逐渐退后，一个更清晰的你才显影出来。这里可以用你约定的 portrait.png，也可以先经过一层处理，让它不是硬切出现，而是从火焰中心慢慢解析出来。这个“看见人”的时刻应该像奖励，也像信任建立的结果。不是用户一上来就得到“脸”，而是先认识你的温度，再看到你的形状。

## 五、网站结构

这个网站的结构不应该以“栏目”来定义，而应该以**交互层**来定义。也就是说，表面看是一个对话界面，内部其实挂着几种不同层次的内容系统。agent 负责调度这些系统，而不是让用户自己面对导航树。

我建议网站的主结构分为五层。

### 1. 门厅层：Agent Landing

这是用户进入后第一眼看到的部分。页面极简，留白很多，中间是火焰 / 信号体，上方或侧边只有极少量的信息，例如你的名字、极简状态字、小小的语言切换、回访识别等。这里最重要的是气氛建立。

这层不应该有密集介绍，也不应该铺卡片墙。它唯一的任务是：

* 建立“这里不是普通主页”的感受
* 让访客立刻理解“应该通过 agent 进入你”
* 让视觉隐喻成立：火焰是你，也是你的 agent，也是你的内核

### 2. 对话层：Conversation as Navigation

这是整站的主层。访客和 agent 对话，内容通过对话组织和展开。也就是说，用户可以直接问：

* 你最近在做什么？
* AuraMate 是什么？
* 你为什么从学术走向命理产品？
* 你做研究和做创作之间是什么关系？
* 你最在意什么？
* 我想快速看你的论文
* 我想知道你的网站为什么这样设计

agent 会根据问题把用户带向不同内容层，并在回复时适度插入可展开的卡片、图片、时间线、项目详情、引用语、链接等。

这里的关键不是“会聊天”，而是**聊天本身就是信息架构**。

### 3. 展开层：Structured Reveal

当用户对某个话题感兴趣时，agent 可以把页面部分切换成“展开视图”。例如用户问 AuraMate，右侧或下方就展开一个非常克制的项目视图；问论文，则出现 research archive；问经历，则出现一条时间线；问审美，则出现视觉碎片和引用。

也就是说，这个网站不是只有聊天框，它是“聊天驱动的动态页面展开”。

这样做的好处是，整个站仍然保持一个统一的主界面，但内容可以因人而异地显现，而不是一次性摊开。

### 4. 深层层：Private Core

这里是最深的内容层。不是所有用户都必须到达，也不是所有用户都需要看到。这里可以包含：

* 肖像显影
* 名字系统：陈姜希 / Jiangxi Chen / Jessy / 陈宇坤
* 丁火 / 伤官生财的诗性锚点
* 那些最不简历化的句子
* 你的转向与困惑
* 真正支撑你继续往前走的愿望

这一层不应该信息密集，而应该非常克制。它像内室，不像展馆。

### 5. 侧门层：Minimal Archive

你明确说不要做两个版本，但 `/minimal` 仍然可以存在，只是它不作为主体验，而是作为一个高效检索侧门。有人只想看论文、邮箱、项目索引、链接，给他一个简洁入口就够了。

主站负责“被看见”，侧门负责“被查找”。

## 六、agent 的角色设计

这个 agent 不能只是“帮你回答问题的机器人”，它必须有非常清楚的角色定位。最准确的定位是：

> **它不是你本人，也不是普通助手。它是“管理你在这个网站上如何被看见”的存在。**

也就是说，它既了解你，也不完全等同于你。它像一个替你守门、替你组织叙事、替你判断进入深度的接口。

这个角色应该同时承担四种功能。

### 第一种功能：门卫

它决定访客最先看到什么，不让整个网站上来就信息溢出。

### 第二种功能：翻译器

它能把你的学术、项目、命理产品、审美表达、人生转向用不同的方式讲给不同的人听。对技术背景的人，它可以更结构化；对投资人，它可以更聚焦创造性与产品；对普通访客，它可以更人性。

### 第三种功能：策展人

它不是机械吐信息，而是组织访客的观看路径。比如先讲 AuraMate，再引向 FateCouncil；或者先从论文切入，再引出“被看见”的工程学。

### 第四种功能：镜子

越深入，它的语言越接近你真正的声音，直到在最深处，agent 慢慢退后，让“你”自己说话。

所以整站不是“你和 agent 并列出现”，而是“agent 先出现，然后把你显影出来”。

## 七、视觉系统

### 1. 视觉气质

这套视觉的核心不是“炫”，而是：

> **冷静的系统界面里，封存着一团活的火。**

你要的高级感来自克制，你要的酷来自“一个活物在极简系统中慢慢显影”。

整体气质应该是：

* 极浅背景，接近雾白或很淡的灰紫
* 大量留白
* 超轻、精密的小字
* 低对比界面
* 中央一个强烈但克制的视觉核心
* 局部微弱仪器感标记和状态字
* 动效非常慢，非常轻，像呼吸而不是炫技

### 2. 核心视觉对象：Flame Core

这个火焰不是装饰，它是网站的心脏。它应该具备四种形态：

#### 形态一：Signal Flame

抽象、微弱、玻璃般、像低频闪烁的生命体。用户初入时使用。

#### 形态二：Inner Flame

更具体的火焰感，内部有温暖颜色，边缘有褐化和湍流，像你说的第二张琉璃火。用于进入对话后。

#### 形态三：Figure in Flame

火焰内部出现半调人像、面部噪声、轮廓线、局部眼睛或嘴的结构。用于进入较深对话。

#### 形态四：Portrait Reveal

火焰和人像短暂重合，再慢慢让位给更清晰的肖像。用于最深层。

### 3. 肖像呈现

你给的图很有参考价值，它告诉我们一个很重要的方向：**人脸不应该是普通头像，而应该是像从系统噪声里被读出来。**

所以肖像出现的方式建议是：

* 前期用半调纹理、粒子、扫描感、灰度点阵
* 不要直接用高保真照片砸脸
* 初次出现时可以是一个不完全稳定的人像
* 随着深度和信任，才逐渐更清楚、更真实

这件事很重要，因为它把“个人网站上的头像”从信息元素变成了叙事事件。

### 4. 配色建议

主色不应该太多，整体以“浅底 + 深灰字 + 火焰少量暖色”为主。

建议组合：

* 背景：雾白、偏灰白、微紫白
* 文本：冷灰、浅黑
* 辅助线：极浅灰线
* 核心发光：琥珀、金橙、浅蓝、微紫
* 深层状态：局部偏棕褐、焦边感

配色要服务“低饱和系统界面 + 局部生命体”这个逻辑。

### 5. 字体与排版

排版要像精密界面，而不是杂志拼贴。建议主文本中文用干净的人文黑体或现代宋黑混用；英文字体可以搭配极简 sans，配一点极细等宽做状态字。

整个页面的文字密度不应大。重要内容通过留白、轻字号、分层对齐来显得高级。

## 八、交互机制

### 1. 两条主状态轴

整个网站可以由两条轴控制：

#### 深度轴 Depth

表示访客进入多深。不是公开告诉用户“你升级了”，而是内部系统状态。

* Depth 0：仅接触到信号
* Depth 1：开始对话
* Depth 2：内容展开
* Depth 3：内核接近
* Depth 4：肖像显影

#### 清晰度轴 Clarity

表示“你”被看见到什么程度。

* 0%：纯火焰
* 25%：模糊人形
* 50%：半调五官
* 75%：较清晰人像
* 100%：肖像 fully revealed

这两条轴不完全等价。比如用户即便没有浏览很多页面，但如果问得非常深，Clarity 可以加快提升。

### 2. Agent 工具循环

你之前就已经认可“agent 两轮工具循环”的方向，这里非常适合继续用。访客提问时，agent 的工作不只是生成回复，而是：

* 判断用户意图
* 调用相应内容模块
* 决定是回答、展示项目、还是推进深层叙事
* 把结构化数据回塞到前端
* 驱动火焰 / 肖像 / 深度状态变化

所以前端能感受到“它不是纯聊天”，而是在边说边调取世界中的材料。这会让站点很有 agent 时代的真实感。

### 3. Returning Visitor

回访机制非常重要。这个网站尤其适合认回头客。一个回访者不应再次从纯火焰开始，而应该出现一种被认出的感觉，例如：

* “你回来了。”
* 火焰比第一次更稳定
* 上次对话主题被轻轻提起
* 深度状态部分保留
* 某些未完成话题重新浮现

这样网站就从一次性浏览变成了关系型界面。

### 4. Voice 驱动

你已经明确希望有 Voice + 振幅驱动视觉，这在这个概念里非常成立。因为“presence”本来就不应该只靠文字。语音输入或语音回应时，火焰的形状、大小、粒子密度、明暗都可以轻微变化，像它真的在呼吸。这样用户会更强烈地感到：这个 agent 不是窗口，它是一个在场的存在。

## 九、内容组织方式

这个网站不适合传统菜单，但内容仍然需要被很好地组织。最好的方式不是按栏目，而是按**agent 可以带出的几条主叙事线**来组织。

### 第一条：研究与系统

这是“别人最容易理解的你”。它包括：

* SJTU 博士
* Deep RL / GNN / 智能运维
* 论文
* 量化研究经历
* 你处理复杂系统的能力

但这里不只是列成果，而是讲：你如何把依赖、不确定性、失效、策略、成本变成结构。

### 第二条：创造与产品

这是“你现在真正想做的你”。它包括：

* AuraMate
* FateCouncil
* MangPai
* 网站本身
* 你如何用工程方法做“被看见”

这里要清楚地传达：你不是在“做一个算命工具”，而是在做个人意义系统的 interface。

### 第三条：记忆与审美

这是“不能被简历表达的部分”。它包括：

* Tears in Rain
* 王家卫
* Blade Runner
* 摄影
* 文字
* 那种关于时间、滞留、雾、雨、失去的感受

这一条不一定单独做成页面，但非常适合被 agent 在对话中慢慢带出。

### 第四条：身体与外部世界

这是“你如何从高强度脑内系统中返回身体”。它包括：

* 山
* 徒步
* 滑雪
* 发呆
* 那种把自己整个展平的休息方式

这一条会让整个网站不只停留在脑和语言，也让“人”更完整。

### 第五条：内核

这里包含那些最深的句子：

* 最怕只剩下简历上的样子
* 最快乐是凌晨三点算法跑通
* 系统可以被优化，人的秩序是用来被温柔打破的
* 想做真正有创造力的事
* 名字与自我命名

这里就是肖像显影最终要落下的地方。

## 十、首页样式建议

首页最重要的不是信息量，而是气场。它可以非常克制。一个比较理想的结构是：

页面几乎全白，只保留极浅噪点和轻微界面感。中央偏上是一团模糊火焰，里面隐约有一点类人形状，但还不明确。左上只有很小的 icon、语言切换、状态字。中央下方是一句极短的开场白，再下面是对话区。

你可以让第一句文案不是自我介绍，而是一个更像“presence 被感知”的句子，例如：

> 这里不是一份完整的简介。
> 你会先遇见一个替我说话的存在。

或者更简洁一点：

> I am not fully visible at first.

再下面是 agent 的第一句欢迎语。它不必非常热情，甚至可以有一点淡淡的距离感，例如：

> 我可以带你认识她。你想从哪一层开始？

这比“你好，我是 Sway，很高兴见到你”更对味。

## 十一、页面不是“聊天框 + 卡片”，而是“对话驱动的舞台”

这一点很关键。你上传的图已经有一个方向：它不是传统 dashboard，而是一个很安静的聊天舞台。你可以继续走这个方向，但要再推进一步：**不要把对话理解成一列消息，而要把它理解成整个页面的导演机制。**

也就是说，对话发生时：

* 中央火焰会变化
* 文字区域会轻轻展开
* 某个项目卡会在一侧浮现
* 某段时间线会被拉出
* 某幅图像会变清晰
* 某个名字会被点亮
* 某些“建议提问”会根据上下文改变

整个界面像一个很安静的舞台，而 agent 通过对话调度舞台。

这会让网站具有真正的“agent 界面感”，而不是只是在页面里塞一个 LLM。

---

## 十二、内容文案风格

文案不能过度卖弄，也不能太职业模板。整体应该是：

* 冷静
* 克制
* 有一点诗性
* 但仍然清楚
* 绝不做廉价神秘感

适合你的文案不是夸张句，而是那种稍微偏硬、又有余味的句子。比如：

* 我研究系统如何被优化，也在学习怎样不把自己优化成一个标准答案。
* 她很擅长做系统喜欢的人。后来开始怀疑，这是不是唯一的活法。
* 这不是一份被一次性读完的简介。
* AuraMate 不是一次性分析工具，而是一种长期被看见的界面。
* 不是每个问题都应该被压缩成最优解。
* 这里有一团小火。它不大，但一直在。

这种语言适合你，也适合这个网站。

## 十三、实现建议

技术上，你完全可以把它做成一个状态驱动的前端系统。核心不是技术难度，而是状态设计要清楚。可以有这样一组核心状态：

* activeTopic：当前对话话题
* depth：进入深度
* clarity：人像清晰度
* presenceMode：signal / flame / figure / portrait
* returningVisitor：是否回访
* lastSessionMemory：上次主题
* revealFlags：是否解锁某些深层内容

视觉对象 Flame Core 可以作为一个独立组件存在，根据这些状态改变。agent 回复时，不只返回文本，还返回：

* 是否需要展开某个模块
* 是否提高 clarity
* 是否改变 presenceMode
* 是否推荐继续向某个话题深入

这样整个网站会形成一个完整的“内容 + 视觉 + 对话”共振系统。

## 十四、最终判断

你的这个网站最重要的转向，就是从“做一个漂亮的个人站”变成了“做一个以 agent 为主角的个人界面”。这是对的，而且非常适合你。

因为你真正想做的不是把自己讲清楚，而是设计一种新的被理解方式。agent 恰好能承担这个角色：它既能保护你，不让一切上来就裸露；又能逐渐把你显影出来，让那个真正的你不是被摆上去，而是被慢慢看见。

所以这份设计档案最后可以用一句话来概括：

> **这不是一个展示陈姜希的网站。
> 这是一个由 agent 引导访客，逐步见到陈姜希的过程。**

而“从火焰到具体的人慢慢清晰”，就是这个过程最准确的视觉母题。
